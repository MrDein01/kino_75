# Сайт АНО «Культура 75»

Готовый PHP-сайт без фреймворков и базы данных. Новости хранятся в `data/news.json`, фотографии загружаются в `uploads/`.

## Что есть в проекте

- Страница «О нас» с позиционированием организации, направлениями деятельности и блоком открытости.
- Страница «Новости» с фильтром по годам с 2025 года и поиском.
- Полная страница новости.
- Админка для добавления/редактирования/удаления новостей.
- Загрузка фото с компьютера: JPG, PNG, WEBP, GIF до 8 МБ.
- Адаптивный дизайн в стиле присланного референса: черный/белый фон, оранжевый полукруг, угловая рамка.

## Вход в админку

Адрес: `/admin/`

Перед публикацией обязательно поменяйте пароль в `includes/config.php`: замените `ADMIN_PASSWORD_HASH` на новый хеш.

Создать хеш можно командой:

```bash
php -r "echo password_hash('НОВЫЙ_ПАРОЛЬ', PASSWORD_DEFAULT), PHP_EOL;"
```

## Как запустить локально

В папке сайта выполните:

```bash
php -S localhost:8000
```

Потом откройте в браузере:

```text
http://localhost:8000
```

## Как добавить новость

1. Откройте `/admin/`.
2. Нажмите «Добавить новость».
3. Заполните:
   - тема новости / заголовок;
   - дата;
   - категория;
   - краткое описание для карточки;
   - полное описание;
   - фото из проводника/загрузок;
   - галочка «Опубликовать на сайте».
4. Нажмите «Сохранить».

## Что писать в новостях для грантовой оценки

Чтобы новости работали как доказательная база, в каждую публикацию желательно добавлять:

- проблему или задачу события;
- целевую аудиторию;
- место и дату проведения;
- количество участников;
- партнеров;
- фотоотчет;
- достигнутые результаты;
- отзывы или цитаты участников;
- дальнейшие планы.

## Публичные документы

Я не положил исходные PDF в публичную папку сайта, чтобы случайно не раскрыть служебные реквизиты из отчетности. Если руководитель разрешит публикацию, положите согласованные файлы в `assets/docs/` и добавьте ссылки на странице `index.php` в блок «Открытость и результаты».

## Развертывание на хостинге

Загрузите все файлы в корень сайта на PHP-хостинге. Проверьте, что папки `data/` и `uploads/` доступны на запись серверу. Для Apache уже добавлены `.htaccess`, закрывающие прямой доступ к `data/` и `includes/`.
